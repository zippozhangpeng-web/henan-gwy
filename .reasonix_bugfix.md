# 好风公考情报站 - Bug修复任务

## 已知问题清单

### Q1. 搜索页默认年份逻辑混乱
- 当前 `app.py` 中 search() 路由: empty year -> '2025', 'all' -> '' 
- 但 `templates/search.html` 的 `<select>` 默认显示与后端不一致
- 用户打开 `/search` 时，下拉框应默认选中"2025年"，且url自动变为 `?year=2025`
- 全部年份选项传 `year=all`，后端识别为无年份筛选
- 在 `templates/search.html` 中使用 `current_year` 变量

### Q2. 进面分显示问题
- `templates/search.html` 的分数列 `{% if pos.min_score %}` 在 min_score=0 时不显示
- 应改为 `{% if pos.min_score is not none and pos.min_score > 0 %}`
- 同时颜色判断用 min_score 而非 avg_score

### Q3. 岗位详情页可能404
- 从搜索页点"情报"按钮跳到 `/position/<id>` 可能404
- 需要确保 `get_position_detail()` 函数健壮

### Q4. 首页推荐岗位
- 首页 `templates/index.html` 的推荐岗位是否正常渲染
- 迷你雷达图是否初始化

### Q5. 数据库启动检查
- `app.py` 的启动检查逻辑: 检测 min_score 数量，不足则自动导入
- 确保 `import_scores.py` 和 `import_pdf_scores.py` 在启动时能正确运行
