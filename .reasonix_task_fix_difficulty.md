# 好风公考情报站 — 上岸难度评分逻辑修正

## 问题描述
上岸难度（difficulty_score）的计算逻辑与另外三个维度（地区优劣、薪酬待遇、发展前景）方向不一致。

**现状（错误）：**
- 进面最低分 > 68 → difficulty_score += 2.0（分越高 = 难度越高）
- 但其他三个维度都是"分数越高 = 对考生越有利"

**要求修正：**
- 四个维度统一语义：**分数越高 = 对考生越有利**
- 上岸难度 = 进面分越低越有利（越容易上岸）→ 难度评分越高
- 进面分越高 → 越难上岸 → 难度评分越低

## 需要改的文件

### 1. `utils/scoring.py` — `calculate_difficulty()` 函数
**反转进面分数的加减方向：**
```python
# 旧(错误)：高进面分 → 难度分更高
if min_score > 68: score += 2.0
elif min_score > 63: score += 1.0
elif min_score < 55: score -= 1.0

# 新(正确)：高进面分 → 越难上岸 → 难度分更低
if min_score > 68: score -= 2.0    # 高分段，难上岸，减分
elif min_score > 63: score -= 1.0  # 中高分段，减分
elif min_score < 55: score += 1.0  # 低分段，易上岸，加分
```

### 2. `utils/scoring.py` — `get_score_detail()` 中难度因子和规则
- **`difficulty_rule`** 字符串改为：
  ```
  "基准线5.0，城市竞争力强→加分(竞争大但岗位好)，招录人数少→减分(难上岸)，进面分高→减分(越难上岸)，学历门槛高→加分(竞争小)，政治面貌有限制→加分(竞争小)"
  ```
- **难度因子数组（difficulty_factors）**：反转进面分数因子的 impact 方向：
  - min_score > 68：原 '+2.0' → '-2.0'，描述改为'高分进面线，难度大'
  - min_score > 63：原 '+1.0' → '-1.0'
  - min_score < 55：原 '-1.0' → '+1.0'，描述改为'低分进面线，相对容易'
- **城市竞争力和招录人数因子不动**（它们已经正确：郑州+2.0表示竞争大但岗位吸引力大，所以整体评分受益）

### 3. `utils/scoring.py` — 难度标签阈值反转
```python
# 旧(按困难程度)：>=8困难, >=6中等, <6容易
# 新(按有利程度)：>=8容易(有利), >=6中等, <6困难(不利)
if difficulty >= 8:
    diff_label = '🟢 容易'
    diff_desc = '上岸难度较低，进面分数线相对友好，性价比较高。'
elif difficulty >= 6:
    diff_label = '🟡 中等'
    diff_desc = '难度适中，有一定竞争压力，认真备考有望上岸。'
else:
    diff_label = '🔴 困难'
    diff_desc = '竞争较激烈，进面分数线较高，建议充分备考。'
```

### 5. `app.py` — 更新 `get_difficulty_label()` 函数
```python
def get_difficulty_label(score):
    if score >= 7:
        return '🟢 容易'
    elif score >= 5:
        return '🟡 中等'
    return '🔴 困难'
```

### 6. 重新计算 `position_scores` 表
修改完 python 逻辑后，执行以更新数据库中的 position_scores 表：
```python
import sqlite3
from utils.scoring import calculate_difficulty, calculate_region, calculate_salary, calculate_prospect

conn = sqlite3.connect('gwy_data.db')
c = conn.cursor()
rows = c.execute('SELECT id, city, system_type, education, recruit_num, experience_requirement, political_status, COALESCE(min_score, 0), unit, district FROM positions').fetchall()
c.execute('DELETE FROM position_scores')
cnt = 0
for row in rows:
    pid, city, st, edu, num, exp, pol, mn, unit, district = row
    pos = {'city': city, 'system_type': st, 'education': edu, 'recruit_num': num, 
           'political_status': pol, 'min_score': mn if mn > 0 else None, 'district': district}
    d = calculate_difficulty(pos)
    r = calculate_region(pos)
    s = calculate_salary(pos)
    p = calculate_prospect(pos)
    ov = round((d + r + s + p) / 4, 1)
    c.execute('INSERT OR REPLACE INTO position_scores (position_id, difficulty_score, region_score, salary_score, prospect_score, overall_score) VALUES (?,?,?,?,?,?)', (pid, d, r, s, p, ov))
    cnt += 1
conn.commit()
conn.close()
print(f'✅ 重新计算 {cnt} 个岗位评分')
```
