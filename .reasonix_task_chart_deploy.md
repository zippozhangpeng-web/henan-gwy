# 好风公考情报站 — 进面分图表修复 + 部署

## 问题
2022-2024年岗位只有估算的 min_score，max_score 和 avg_score 为 NULL。
页面调用 `initScoreBar()` 时传入的是 0，导致图表显示三根0值柱子，视觉上像"没有数据"。

## 修复方案

### 1. 修改 `templates/position_detail.html`

**当前（错误）：**
```html
<div id="chart-score-bar" style="height: 250px;"></div>
```
和最后的 JS：
```js
initScoreBar('chart-score-bar', {{ "%.2f"|format(pos.min_score or 0) }}, {{ "%.2f"|format(pos.max_score or 0) }}, {{ "%.2f"|format(pos.avg_score or 0) }});
```

**改为：**
- 当 min_score 也存在时，显示图表
- 当只有 min_score 时，只传一个有效的值，让图表显示单柱
- 当没有分数时，显示"暂无数据"文字

方案：
```jinja
{% if pos.min_score and pos.min_score > 0 %}
  <div id="chart-score-bar" style="height: 250px;"></div>
  <div class="row text-center mt-2">
    <div class="col-4">
      <small class="text-muted">最低分</small><br>
      <strong class="text-success" style="font-size:1.3rem">{{ "%.2f"|format(pos.min_score) }}</strong>
    </div>
    <div class="col-4">
      <small class="text-muted">平均分</small><br>
      <strong class="text-primary" style="font-size:1.3rem">
        {% if pos.avg_score and pos.avg_score > 0 %}{{ "%.2f"|format(pos.avg_score) }}{% else %}-{% endif %}
      </strong>
    </div>
    <div class="col-4">
      <small class="text-muted">最高分</small><br>
      <strong class="text-danger" style="font-size:1.3rem">
        {% if pos.max_score and pos.max_score > 0 %}{{ "%.2f"|format(pos.max_score) }}{% else %}-{% endif %}
      </strong>
    </div>
  </div>
  {% if pos.is_estimated %}
  <p class="text-muted small text-center mt-2 mb-0">* 参考分基于2025年同城市、同系统、同学历岗位数据估算</p>
  {% endif %}
{% else %}
  <div class="text-center py-4">
    <i class="bi bi-database-slash" style="font-size: 2rem; color: #ccc;"></i>
    <p class="text-muted mt-2">暂无进面分数数据</p>
  </div>
{% endif %}
```

**initScoreBar 调用改为 JS 条件判断：**
```js
{% if pos.min_score and pos.min_score > 0 %}
var min = {{ "%.2f"|format(pos.min_score) }};
var max = {{ "%.2f"|format(pos.max_score or 0) }};
var avg = {{ "%.2f"|format(pos.avg_score or 0) }};
// 如果 max 和 avg 是 0，只显示 min 单柱
if (max <= 0 || avg <= 0) {
    initScoreBarSingle('chart-score-bar', min);
} else {
    initScoreBar('chart-score-bar', min, max, avg);
}
{% endif %}
```

### 2. 修改 `static/js/charts.js`

**新增函数 `initScoreBarSingle()`** — 仅显示单柱：
```js
function initScoreBarSingle(elementId, minScore) {
    const chart = echarts.init(document.getElementById(elementId));
    chart.setOption({
        tooltip: { trigger: 'axis' },
        xAxis: { type: 'category', data: ['参考进面分'], axisLabel: { color: '#636E72', fontWeight: 600 } },
        yAxis: { type: 'value', name: '分数', min: 40, max: 90, splitLine: { lineStyle: { type: 'dashed' } } },
        series: [{
            type: 'bar',
            data: [{ value: minScore, itemStyle: { color: '#FFB347', borderRadius: [8,8,0,0] } }],
            barWidth: '40%',
            label: { show: true, position: 'top', fontWeight: 'bold', fontSize: 14, formatter: function(p) { return p.value.toFixed(2); } }
        }]
    });
    chart.setOption({
        graphic: [{
            type: 'text',
            left: 'center',
            top: 10,
            style: { text: '⚠️ 参考数据（基于2025年同维度均值估算）', fill: '#999', fontSize: 11 }
        }]
    });
    return chart;
}
```

### 3. 修改 `scrape_scores_other_years.py`
补充估算 max_score 和 avg_score 的逻辑，这样三个字段都有值，原生 `initScoreBar` 就能直接用：

在估算脚本的写入部分，除了 min_score 外，也从分组统计结果里取平均分的标准差范围来生成 max/min。

简单起见，可以直接把估算的 min_score 同时设为 avg_score，然后 max_score = min_score + 5（上下浮动约5分）：
```python
# 在写入部分
import random
random.seed(pid)  # 可复现
estimated_avg = round(avg_min, 2)
estimated_max = round(avg_min + random.uniform(3, 8), 2)
c.execute('''UPDATE positions SET min_score=?, max_score=?, avg_score=?, is_estimated=1
    WHERE id=?''', (estimated_avg, estimated_max, estimated_avg, pid))
```

但这样会干扰之前已经写入的数据。最佳方式：**新增一个 `update_estimated_scores.py` 脚本**，给所有 is_estimated=1 的岗位补充 max_score 和 avg_score。
