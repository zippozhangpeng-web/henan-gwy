## 问题一：四维评价缺说服力（核心问题）

### 现状
- `utils/scoring.py` 的 `get_score_detail()` 为每个维度生成详细描述文本（如difficulty_detail、region_detail）
- 但 `templates/position_detail.html` 只展示了最终分数 + 一段笼统文字，用户看不到"这个分数到底怎么算出来的"
- 首页推荐卡片只展示综合评分数字和迷你雷达，没有评分依据

### 改造目标
在岗位详情页（position_detail.html）的四维评分卡片中，增加**评分依据/计算明细**，让每个打分有据可查。

### 具体需求

#### 1. 修改 `utils/scoring.py` 的 `get_score_detail()` 函数
在每个维度中增加 `factors` 数组，展示评分因子明细：

**上岸难度（difficulty）示例：**
```python
difficulty_factors = [
    {'name': '城市竞争力', 'value': f'{city}是省会城市', 'impact': f'+{factor_score}'},
    {'name': '招录人数', 'value': f'仅招{recruit_num}人', 'impact': f'+1.5'},
    {'name': '进面分数', 'value': f'最低{min_score}分', 'impact': f'+{factor_score}'},
    {'name': '学历要求', 'value': education, 'impact': f'{factor_score}'},
    {'name': '政治面貌', 'value': political, 'impact': f'-0.5'},
    {'name': '基础分', 'value': '5.0（基准线）', 'impact': '+5.0'},
]
```

每个维度都增加 `{dimension}_factors` 数组和一个计算规则说明文字 `{dimension}_rule`，例如：
- `difficulty_factors`: `[{'name', 'value', 'impact'}]` — 每个因子的具体数值和影响分值
- `difficulty_rule`: 简短的一句话规则说明

**地区优劣（region）增加 factors：**
```python
region_factors = [
    {'name': 'GDP排名', 'value': '全省第X位', 'impact': f'{score}'},
    {'name': '人均薪资', 'value': 'X元/月', 'impact': '-'},
    {'name': '房价水平', 'value': 'X元/㎡', 'impact': '-'},
    {'name': '是否为省会', 'value': '是/否', 'impact': '+' if capital else '-'},
]
```

**薪酬待遇（salary）增加 factors：**
```python
salary_factors = [
    {'name': '城市平均薪资', 'value': 'X元/月', 'impact': f'{base_score}'},
    {'name': '系统加成', 'value': f'公安系统+0.8', 'impact': f'+{bonus}'},
    {'name': '层级加成', 'value': f'省直+0.5', 'impact': f'+{bonus}'},
]
```

**发展前景（prospect）增加 factors：**
```python
prospect_factors = [
    {'name': '部门权重', 'value': f'{sys_type}', 'impact': f'{base_score}'},
    {'name': '层级加成', 'value': f'{district}', 'impact': f'+{bonus}'},
    {'name': '城市加成', 'value': f'{city}', 'impact': f'+{bonus}'},
]
```

**注意：** 这些因子数据必须从真实数据库中该岗位的数值计算而来，不能写死。

## 问题二：进面分数不全

### 现状
数据库中有5年数据（2022-2026），但进面分数只在2025年部分存在：
- 2022年：4821岗位，分数=0
- 2023年：5351岗位，分数=0
- 2024年：6231岗位，分数=0
- 2025年：6167/7629有效分 + 714空 + 748异常值(0.0或>100)
- 2026年：7509岗位，分数=0（2026年考试可能面试尚未开始）

### 任务

#### A. 清理2025年异常数据
`import_scores.py` 和 `import_pdf_scores.py` 使用了 `COALESCE(min_score, ?)` 导致部分异常值被写入（min_score=0.0 或 500+ 等明显错误）。

编写一个清理脚本 `clean_scores.py`：
1. 找出所有 min_score=0.0 的岗位，设为 NULL（无数据比假数据好）
2. 找出所有 min_score>100 的岗位，检查是否是笔试总分而非面试分，如果是则删除
3. 找出所有 avg_score 异常（>100）的岗位，同样清理
4. 清理完后统计覆盖率

#### B. 获取2022-2024年进面分数
进面分数通常在面试确认公告附件中发布。官方发布渠道：
- 河南省人事考试中心：http://www.hnrsks.com/
- 各地市人社局官网
- 各招录单位官网

**但注意：** 这台机器的网络通过 ClashX 代理，DNS 走假IP模式（198.18.x.x），所以 web_fetch/requests 可能被 SSRF 防护拦截。用 `curl` 或 `requests` 时可能需要设置代理。

**编写 `scrape_scores_other_years.py`，功能：**

策略一：从官方渠道批量搜索下载：
1. 搜索 `site:henan.gov.cn 2024 河南省考 面试确认名单` 等关键词
2. 下载附件（xls/xlsx/pdf），解析其中的职位代码+总成绩
3. 按年份写入数据库

策略二：如果策略一的直链下载困难，尝试从中公/华图等培训机构网站抓取汇总页面。

如果网络限制导致无法下载，则：
4. 在数据库中给2022-2024年岗位生成**参考分数**：
   - 基于同城市同系统同学历已有2025年数据的平均分推算
   - 生成的分数标注为`is_estimated=1`字段（需要新增schema）
   - 在页面上显示时标注"参考分"标签

#### C. 更新进面分显示逻辑
修改 `templates/position_detail.html` 和 `templates/search.html`：
- 当 min_score IS NULL 时显示"暂无数据"而非"-"
- 如果有 `is_estimated` 标记，显示"参考分"标签（橙色badge）
- 搜索页排序支持按"参考分"排序
